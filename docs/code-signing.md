# Подпись кода и предупреждение Windows SmartScreen

## Проблема (замечание #1)

При запуске `.exe` Windows показывает «Windows защитила ваш компьютер» /
«Неизвестный издатель» (Microsoft Defender SmartScreen). Причина — установщик
**не подписан** доверенным сертификатом. Это не баг лаунчера: так Windows ведёт
себя с любым неподписанным приложением от неизвестного издателя. Полностью
убрать предупреждение можно только подписью кода — в самом коде это не решается.

## Варианты решения (по возрастанию стоимости/эффекта)

1. **Никак не подписывать (текущее состояние).**
   Пользователю нужно нажать «Подробнее» → «Выполнить в любом случае».
   Стоит описать этот шаг в инструкции для игроков (со скриншотом).
   Бесплатно, но отпугивает часть пользователей.

2. **Azure Trusted Signing** (рекомендуется как дешёвый «настоящий» вариант).
   ~2 $/мес для индивидуальных разработчиков/малого бизнеса. Microsoft хранит
   ключ в облаке, подпись делается из CI. Tauri-action это поддерживает.
   Требование: аккаунт существует не менее ~3 лет (для individual-валидации).

3. **Обычный OV code-signing сертификат** (Sectigo/DigiCert и т.п.).
   ~70–200 $/год. Подпись есть, но репутация SmartScreen набирается не сразу —
   первые загрузки всё равно могут показывать предупреждение, пока не накопится
   статистика по сертификату.

4. **EV code-signing сертификат.**
   ~250–400 $/год + аппаратный токен/HSM. Даёт **мгновенную** репутацию
   SmartScreen — предупреждение не показывается с первого релиза. Самый дорогой,
   но самый «чистый» для игроков вариант.

## Как подключить подпись в CI (когда будет сертификат)

`tauri-action` (`.github/workflows/release.yml`) умеет подписывать Windows-сборки.
Кратко по вариантам:

- **Azure Trusted Signing** — задать переменные окружения шага сборки:
  `AZURE_TENANT_ID`, `AZURE_CLIENT_ID`, `AZURE_CLIENT_SECRET` и в
  `tauri.conf.json → bundle.windows` настроить `signCommand` через
  `trusted-signing-cli`, либо использовать готовый action azure/trusted-signing.
- **Сертификат (.pfx)** — секреты `WINDOWS_CERTIFICATE` (base64 .pfx) и
  `WINDOWS_CERTIFICATE_PASSWORD`; в `tauri.conf.json → bundle.windows`
  указать `certificateThumbprint`/`signCommand`. Tauri подпишет и установщик,
  и `.exe`.

> Подпись кода (Authenticode) — это **другая** подпись, не путать с ключом
> Tauri Updater (`TAURI_SIGNING_PRIVATE_KEY`): updater-ключ проверяет
> подлинность апдейтов внутри лаунчера, а Authenticode убирает предупреждение
> Windows при первом запуске. Нужны обе.

## Что уже сделано в коде

- В `tauri.conf.json` заданы `publisher`, `copyright`, описание и NSIS-настройки
  (`installMode: currentUser` — установка без прав администратора, что снижает
  число предупреждений UAC). Это не убирает SmartScreen, но делает установщик
  опрятнее и корректно подписывается при добавлении сертификата.
