# Подпись manifest.json (minisign)

Защита от подмены модпака: лаунчер тянет `manifest.json` с GitHub Pages, и вся
цепочка целостности раньше сводилась к «доверяем TLS + GitHub». Теперь манифест
подписывается **minisign** (Ed25519), а лаунчер проверяет подпись перед разбором.
Per-file SHA-256 внутри манифеста подпись накрывает транзитивно.

- **Формат** — minisign (тот же, что у автообновления Tauri), prehashed.
- **Подпись** делает сборщик (`builder/`) → кладёт `manifest.json.minisig` рядом с
  `manifest.json` в `dist/` → CI `krp-mod` деплоит на Pages.
- **Проверка** — `manifest::fetch_manifest` в лаунчере: качает `.minisig`, сверяет
  подпись публичным ключом `config::MANIFEST_PUBKEY`.
- **Ключ отдельный** от ключа автообновления (изоляция компрометации).

## Fail-safe: проверка включается только когда задан публичный ключ

- `config::MANIFEST_PUBKEY` **пуст** → лаунчер подпись НЕ проверяет (текущее
  поведение, ничего не ломается).
- `config::MANIFEST_PUBKEY` **задан** → лаунчер ТРЕБУЕТ валидную подпись
  (fail-closed): нет/битая подпись → ошибка, установка/запуск отменяются.
- Сборщик: нет секрета `KRP_MANIFEST_SECRET_KEY` → манифест не подписывается
  (предупреждение). Есть секрет → пишет `manifest.json.minisig`.

## Порядок раскатки (ВАЖНО — иначе сломаются все установки)

1. **Сгенерировать ключ** (локально, разово) — см. ниже.
2. **Задать секреты в репозитории `Kingdom-RP/krp-mod`**:
   - `KRP_MANIFEST_SECRET_KEY` — содержимое `.key`-файла целиком;
   - `KRP_MANIFEST_SECRET_KEY_PASSWORD` — пароль ключа (если ключ с паролем; иначе
     секрет не создавать).
3. **Запустить публикацию `krp-mod`** (push/`workflow_dispatch`) и убедиться, что
   `https://kingdom-rp.github.io/krp-mod/manifest.json.minisig` отдаётся (HTTP 200).
4. **Только теперь** вписать публичный ключ в `config::MANIFEST_PUBKEY`
   (строка `RWR...` — вторая строка `.pub`-файла) и выпустить релиз лаунчера.

Если сделать наоборот (сначала ключ в лаунчер, потом подпись в CI) — лаунчеры
старых пользователей начнут падать на проверке подписи отсутствующего `.minisig`.

## Генерация ключа

Любым minisign-совместимым инструментом. Например, `rsign2` (чистый Rust):

```sh
cargo install rsign2
# с паролем (рекомендуется):
rsign generate -p krp-manifest.pub -s krp-manifest.key
# без пароля (проще для CI, но ключ-файл = доступ к подписи):
rsign generate -W -p krp-manifest.pub -s krp-manifest.key
```

- `krp-manifest.key` → секрет `KRP_MANIFEST_SECRET_KEY` (всё содержимое файла).
  Пароль (если есть) → `KRP_MANIFEST_SECRET_KEY_PASSWORD`. **Не коммитить.**
- `krp-manifest.pub` → вторую строку (`RWR...`) вписать в
  `src-tauri/src/config.rs` → `MANIFEST_PUBKEY`.

Проверено: `minisign` 0.9 (подпись в сборщике) и `minisign-verify` 0.2 (проверка
в лаунчере) совместимы, prehashed-подпись верифицируется, подделка отклоняется.

## Ротация ключа

Публичный ключ вшит в бинарь лаунчера → смена ключа требует релиза лаунчера
(как и у автообновления). План: сгенерить новый ключ → обновить секрет в `krp-mod`
→ дождаться публикации нового `.minisig` → обновить `MANIFEST_PUBKEY` → релиз.
